# OpenCoder 能力地图

OpenCoder 当前提供以下用户/调用方可感知的能力。每项链接到对应的业务文档或逻辑模块。

## 能力组

- **会话恢复**：CLI `--session <id>` / `--continue` / `--fork`，跨进程从 libsql 重建历史；title 由 small_model 异步生成。详见 [agents/session](../agents/session/index.md)。
- **Session 二进制导出/导入**：`opencode session export <id> -o <file>` 导出 session（含 subagent 树）为 `.opencode` 二进制文件（`OPENCODR` magic）；`opencode session import <file>` 幂等导入到新环境，`--session <id>` 可继续执行。不导出 Config（API key 安全）。详见 [agents/store](../agents/store/index.md) / [bundle changelog](changelog/2026-07-06/compact-clear-display-session-bundle.md)。
- **模型与压缩配置化**：`provider/model`、`small_model`、`context_limit`、`max_tokens`、`reasoning_effort`、`compaction.{auto,context_threshold,reserved,tail_turns,buffer}` 全可经 opencode.json / 环境变量 / CLI flag 配置；压缩首轮即可由 token 估算触发。详见 [agents/session](../agents/session/index.md)、[agents/core](../agents/core/index.md)、[压缩提示词+token 追踪修复](changelog/2026-07-11/compaction-prompt-and-token-tracking.md)。
- **steer / followup（两段式 delivery）**：运行中的会话可在 turn 边界吸收 steer（重置步数配额），idle 时消费恰好一条 queue。HTTP `POST /prompt` 带 `delivery`。详见 [agents/session](../agents/session/index.md)。
- **Web 会话操作**：HTTP CRUD + SSE 事件流（replay+live）+ 运行时 agent/model 切换 + interrupt。详见 [agents/web](../agents/web/index.md)。
- **高性能本地存储**：libsql 嵌入 + WAL，并发读写；`Store` trait 为切换其它 Rust SQLite 实现留口子。详见 [agents/store](../agents/store/index.md)。
- **glm5.2 端到端（深度契约）**：`scripts/e2e_glm.py`（`scripts/e2e-glm.sh` 为薄 wrapper）用真 glm5.2 验证**业务契约**而非表面标记。深度观测面为 `opencode session show <id> --json`（meta + 全 ContentBlock + subagent_tasks，解耦存储内部）。覆盖 12 项：E7/E9 reasoning_effort/interleaved_thinking 显示、E1/E6 write+bash 写游戏并编译（含「工具真被调用」日志断言）、E2 `--continue` 恢复**且加载了首轮上下文**（历史含 turn-1 内容）、E5 `--fork` **复制消息且原 session 角色序列/计数不变**、E3/E3b 压缩自动触发 **+ 摘要引用实际工作 + 压缩后上下文可用**、E4 subagent 派遣 **+ DB 跟踪到终态 + 父答案引用调查**、E8 bundle 导出/导入 **roundtrip 完整性（消息计数相等 + 文本内容相同 + 角色序列相同）**、E10 plan agent **只读（无法创建文件）**、E11 web **两段式 delivery（steer 即时 + queue idle 消费，两个 prompt 都投递）**。模型依赖项用 soft-skip；确定性存储契约（fork/bundle/resume/plan 只读）用 hard assert。注：交错思考的 reasoning_content 真实回传契约仅在强制该行为的模型（DeepSeek-V4）上可完整验证；持久化逻辑由集成测试 `session/tests/interleaved_thinking.rs` 覆盖。
- **TUI**：3-region 布局（body / composer / 底部合并 status：model + agent + dir + ctx%）+ 手动 scrollbar（替代 ratatui ScrollbarState，短内容对齐正确）+ 自动跟随 + 运行中 braille spinner 动画 + 可见 bar 光标 + 实时上下文百分比 + **subagent 可折叠 + ctx 切换 + 计数**（`⇲sub:{running}/{total}` 徽章；点击表头展开/折叠，再点进入子会话 ctx 视图，Esc 返回）+ steer/followup（Enter=steer / Tab=follow-up）+ 双击 Esc 硬中止 + thinking 默认折叠（点击表头展开/收起） + say: 助手标识 + **Shift+Tab 切 plan/act**（Alt+Tab / Ctrl+T fallback）+ plan→act 手动切换后自动执行 + skill 模糊匹配（`$` 选择器）+ `/compact` 手动压缩 + 每 session 独立状态（`/task` 切换保存/恢复 chat+scroll+history）。快捷键：Shift+Tab 切 plan/act、Ctrl+D 退出、Esc 关 help。详见 [迭代三](changelog/2026-07-05/iteration3-tui-overhaul.md) / [迭代四](changelog/2026-07-05/iteration4-tui-ux-hardabort.md) / [subagent 重构](changelog/2026-07-05/subagent-and-tool-refactor.md) / [plan-act 手动切换](changelog/2026-07-06/plan-act-handoff-compact.md) / [压缩+滚动条+subagent 修复](changelog/2026-07-09/)。
- **Skill 选择（TUI `$`）**：空 composer 输入 `$` 弹出技能选择器，扫描 `~/.opencoder/skills`（识别 `<name>.md` 与 `<name>/SKILL.md`，可选 `---` frontmatter 的 `name`/`description`）。↑/↓ 移动、键入即过滤、Enter 激活、Esc 取消。激活后 skill 正文以 `## Active skill` 段注入系统提示末尾（最高优先级），整会话生效，状态栏显示 `skill:<name>`；再次选择替换，已激活时首行「✕ clear」清除。详见 [agents/core](../agents/core/index.md)、[agents/session](../agents/session/index.md) / [skill-picker changelog](changelog/2026-07-05/skill-picker.md)。
- **模型配置（TUI `/model`）**：空 composer 输入 `/` 弹出 slash 命令选择器（`/task` 切会话、`/model` 配模型、`/compact` 手动压缩对话历史）。`/` 与 `/model` 弹窗以输入框顶边为底锚渲染下拉浮层（非屏幕居中）。`/model` 模态可编辑 model / base_url / api_key（掩码 `sk-****1234`，编辑不回显）/ reasoning_effort（off↔low↔medium↔high 四档）/ compaction.context_threshold；**Enter = 确认当前值并推进下一字段**，连续 Enter 到 `[Save]` 提交；`↑/↓` 在文本字段间导航、在 Reasoning/Threshold 上调值。保存经 `Config::save` 持久化到 opencode.json（项目优先）并热重载——`UiCmd::ReloadConfig` 在 turn 边界替换 `session.config`/`session.model` 并重建 `ChatClient`。状态栏 model 旁显示 `·high` 思考深度徽标。详见 [agents/core](../agents/core/index.md)、[agents/llm](../agents/llm/index.md) / [model-command changelog](changelog/2026-07-05/model-command-and-reasoning-effort.md) / [plan-act + /compact](changelog/2026-07-06/plan-act-handoff-compact.md)。
- **测试规则与覆盖**：[rules/](../rules/) 目录建立强制测试规则（每功能必有测试 + 迭代回归 gate + 测试分层）；workspace 测试 296 个，覆盖 LLM 流式原语 / CLI（含 `session show --json` 深度观测面）/ 工具 / subagent / bash_guard / TUI / Web（HTTP 契约 + drain 生命周期）/ prompt / store CRUD / bundle 导出导入 / fork / interleaved thinking。详见 [迭代四 changelog](changelog/2026-07-05/iteration4-test-coverage-rules.md)。
